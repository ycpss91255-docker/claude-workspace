description: All code, documentation, README, section headers must not use emoji
type: feedback
---

Do not use emoji anywhere — code, documentation, README files, section headers, commit messages.

**Why:** User preference for clean, professional text without emoji decorations.

**How to apply:** Before writing or editing any file, ensure no emoji characters are included. When reviewing existing files, flag and remove any emoji found.
