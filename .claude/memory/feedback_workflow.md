使用者對所有開發工作的嚴格要求：

## Git 工作流程
- 在有 .git 的 Workspace 中，處理 BUG 或更新功能時**必須開新 branch**
- 上傳 GitHub 時**必須使用 PR**，不可直接 push main

## TDD 開發導向
- 嚴格 TDD：先寫失敗測試 → 實作讓測試通過 → 重構
- 不可以先寫實作再補測試

## 測試覆蓋率
- 覆蓋率要求 **100%**
- 必須產出覆蓋率報表

## 測試文件
- 有新增測試時，必須更新名為 **TEST.md** 的測試項目文件

## Code Style
- 以 **Google Style** 為主

## 文件同步
- 修改完程式碼後，必須同步更新 README 等敘述檔案
- 確認 README 等敘述檔案是否過時
- 確認是否有多餘的檔案需要清除

**Why:** 使用者明確要求嚴格遵守這些規範，確保程式碼品質與文件一致性。

**How to apply:** 每次開發工作都按此清單逐項執行，不可跳過任何步驟。
